## 1. 维度概念辨析

### 核心问题：seq_len vs emb_dim vs context_length

```
┌─────────────────────────────────────────────────────────────────────┐
│                        GPT 数据流维度图                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  输入 token IDs:    [batch_size, seq_len]                          │
│                     [2, 1024]                                       │
│                          │                                          │
│                          ▼                                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  tok_emb + pos_emb                                          │   │
│  │  [batch_size, seq_len, emb_dim]                             │   │
│  │  [2, 1024, 768]                                             │   │
│  │       ↑        ↑        ↑                                   │   │
│  │    batch    序列长度   嵌入维度                              │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                          │                                          │
│                          ▼                                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  Transformer Blocks                                         │   │
│  │  [2, 1024, 768] → [2, 1024, 768]                           │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                          │                                          │
│                          ▼                                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  out_head (Linear)                                          │   │
│  │  [2, 1024, 768] → [2, 1024, 50257]                         │   │
│  │                              ↑                              │   │
│  │                          vocab_size                         │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 关键区分

| 概念 | 含义 | 典型值 | 所在维度 |
|------|------|--------|----------|
| `seq_len` | 输入序列有多少个 token | 1~1024（可变） | 第 2 维 |
| `emb_dim` | 每个 token 的向量维度 | 768（固定） | 第 3 维 |
| `context_length` | 最大允许的序列长度 | 1024（固定） | 配置参数 |
| `vocab_size` | 词表大小 | 50257（固定） | 输出维度 |

**记忆口诀**：
> seq_len 是"有多少个词"，emb_dim 是"每个词多少维"

## 2. Embedding 层详解

### tok_emb 和 pos_emb 的维度与相加

```
┌─────────────────────────────────────────────────────────────────────┐
│                    Embedding 层工作原理                             │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  tok_emb 权重: [vocab_size, emb_dim] = [50257, 768]                │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  词 ID 0:   [0.1, 0.2, ..., 0.3]  ← 768 维                  │   │
│  │  词 ID 1:   [0.4, 0.5, ..., 0.6]  ← 768 维                  │   │
│  │  ...                                                        │   │
│  │  词 ID 50256: [...]               ← 768 维                  │   │
│  └─────────────────────────────────────────────────────────────┘   │
│         ↓ 根据 token ID 查表                                        │
│                                                                     │
│  pos_emb 权重: [context_length, emb_dim] = [1024, 768]             │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  位置 0:   [0.1, 0.2, ..., 0.3]  ← 768 维                   │   │
│  │  位置 1:   [0.4, 0.5, ..., 0.6]  ← 768 维                   │   │
│  │  ...                                                        │   │
│  │  位置 1023: [...]                ← 768 维                   │   │
│  └─────────────────────────────────────────────────────────────┘   │
│         ↓ 根据位置索引查表                                          │
│                                                                     │
│  ════════════════════════════════════════════════════════════════  │
│                                                                     │
│  输入: "我 爱 北京 天安门" (seq_len=4)                              │
│                                                                     │
│  tok_emb 输出:                                                      │
│  ┌────────┬────────┬────────┬────────┐                             │
│  │  "我"  │  "爱"  │ "北京" │"天安门"│                             │
│  │ 768维  │ 768维  │ 768维  │ 768维  │  ← 词义信息                 │
│  └────────┴────────┴────────┴────────┘                             │
│       +        +        +        +                                  │
│  pos_emb 输出:                                                      │
│  ┌────────┬────────┬────────┬────────┐                             │
│  │ pos_0  │ pos_1  │ pos_2  │ pos_3  │                             │
│  │ 768维  │ 768维  │ 768维  │ 768维  │  ← 位置信息                 │
│  └────────┴────────┴────────┴────────┘                             │
│       ↓        ↓        ↓        ↓                                  │
│  ┌────────┬────────┬────────┬────────┐                             │
│  │"我"+p0 │"爱"+p1 │"北京"+p2│"天安门"+p3│                          │
│  │ 768维  │ 768维  │ 768维  │ 768维  │  ← 词义+位置                │
│  └────────┴────────┴────────┴────────┘                             │
│                                                                     │
│  最终输出: [batch, seq_len=4, emb_dim=768]                         │
│           维度不变！是相加，不是拼接                                │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**关键点**：
- tok_emb 和 pos_emb 输出维度必须相同（都是 768）
- 两者是**逐元素相加**，不是拼接
- 每个位置都有自己的 position embedding

## 3. 模型参数操作

### parameters() vs named_parameters()

```
┌─────────────────────────────────────────────────────────────────────┐
│              parameters() vs named_parameters()                     │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│                    内存中的参数对象                                  │
│                    ┌─────────────────┐                              │
│                    │ layer1.weight   │  ← 同一个对象                │
│                    │ [8, 4] tensor   │                              │
│                    └─────────────────┘                              │
│                           ↑     ↑                                   │
│                           │     │                                   │
│              ┌────────────┘     └────────────┐                      │
│              │                               │                      │
│     ┌────────┴────────┐           ┌─────────┴─────────┐            │
│     │  parameters()   │           │ named_parameters()│            │
│     │  返回: tensor   │           │ 返回: (name, tensor)│           │
│     │  只有值         │           │ 名字 + 值          │           │
│     └─────────────────┘           └───────────────────┘            │
│                                                                     │
│  ════════════════════════════════════════════════════════════════  │
│                                                                     │
│  使用场景对比:                                                      │
│                                                                     │
│  ✅ parameters() 用于:                                              │
│     optimizer = Adam(model.parameters(), lr=0.001)                 │
│     → 优化器只需要参数值                                            │
│                                                                     │
│  ✅ named_parameters() 用于:                                        │
│     for name, param in model.named_parameters():                   │
│         if 'encoder' in name:                                      │
│             param.requires_grad = False  # 冻结特定层              │
│     → 需要根据名字筛选参数                                          │
│                                                                     │
│  ════════════════════════════════════════════════════════════════  │
│                                                                     │
│  它们返回的是同一个对象吗？ → ✅ 是的！                              │
│                                                                     │
│  param_only = list(model.parameters())[0]                          │
│  name, param_named = list(model.named_parameters())[0]             │
│                                                                     │
│  param_only is param_named  # True - 同一个对象                    │
│  id(param_only) == id(param_named)  # True - 内存地址相同          │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

### 参数计数：为什么 shape[0]*shape[1] 会越界？

```
┌─────────────────────────────────────────────────────────────────────┐
│              参数维度不都是 2D！                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  GPT 模型中的参数:                                                  │
│                                                                     │
│  2D 参数 (矩阵):                                                    │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  tok_emb.weight:  [50257, 768]  ✅ 有 shape[0] 和 shape[1]  │   │
│  │  linear.weight:   [768, 768]    ✅ 有 shape[0] 和 shape[1]  │   │
│  │  out_head.weight: [50257, 768]  ✅ 有 shape[0] 和 shape[1]  │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  1D 参数 (向量):                                                    │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  ln.weight (scale): [768]       ❌ 只有 shape[0]            │   │
│  │  ln.bias (shift):   [768]       ❌ 只有 shape[0]            │   │
│  │  linear.bias:       [768]       ❌ 只有 shape[0]            │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  ════════════════════════════════════════════════════════════════  │
│                                                                     │
│  ❌ 错误写法:                                                       │
│  for param in model.parameters():                                  │
│      total += param.shape[0] * param.shape[1]  # 1D 参数会越界！   │
│                                                                     │
│  ✅ 正确写法:                                                       │
│  for param in model.parameters():                                  │
│      total += param.numel()  # 适用于任意维度                      │
│                                                                     │
│  numel() 的计算逻辑:                                                │
│  [50257, 768] → 50257 × 768 = 38,597,376                          │
│  [768]        → 768                                                │
│  [4, 8, 8]    → 4 × 8 × 8 = 256                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 4. LayerNorm 归一化

### LayerNorm 对哪个维度归一化？

```
┌─────────────────────────────────────────────────────────────────────┐
│              LayerNorm 归一化方向                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  输入: [batch=2, seq_len=4, emb_dim=768]                           │
│                                                                     │
│  Batch 0:                                                           │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │ 位置0: [x₁, x₂, x₃, ..., x₇₆₈] ←──────────────────────────┐  │ │
│  │ 位置1: [x₁, x₂, x₃, ..., x₇₆₈] ←─ 对这 768 维独立归一化 ─┤  │ │
│  │ 位置2: [x₁, x₂, x₃, ..., x₇₆₈] ←──────────────────────────┤  │ │
│  │ 位置3: [x₁, x₂, x₃, ..., x₇₆₈] ←──────────────────────────┘  │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  Batch 1:                                                           │
│  ┌───────────────────────────────────────────────────────────────┐ │
│  │ 位置0: [...] ← 独立归一化                                     │ │
│  │ 位置1: [...] ← 独立归一化                                     │ │
│  │ 位置2: [...] ← 独立归一化                                     │ │
│  │ 位置3: [...] ← 独立归一化                                     │ │
│  └───────────────────────────────────────────────────────────────┘ │
│                                                                     │
│  共 2 × 4 = 8 次独立的归一化操作                                    │
│                                                                     │
│  ════════════════════════════════════════════════════════════════  │
│                                                                     │
│  归一化公式 (对每个位置的 768 维向量):                              │
│                                                                     │
│  μ = (1/768) Σᵢ xᵢ           # 768 维的均值                        │
│  σ = √[(1/768) Σᵢ (xᵢ-μ)²]   # 768 维的标准差                      │
│  x̂ᵢ = (xᵢ - μ) / σ           # 标准化                              │
│  yᵢ = γ · x̂ᵢ + β             # 缩放和平移 (可学习参数)             │
│                                                                     │
│  ════════════════════════════════════════════════════════════════  │
│                                                                     │
│  LayerNorm vs BatchNorm:                                            │
│                                                                     │
│  LayerNorm:  对 emb_dim (最后一维) 归一化                          │
│              ←─────────────────→                                    │
│                   768 维                                            │
│                                                                     │
│  BatchNorm:  对 batch 维度归一化                                    │
│              ↓                                                      │
│              ↓                                                      │
│              ↓                                                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**关键点**：LayerNorm 对每个词的 768 维 embedding 独立归一化，不跨词、不跨 batch。

## 5. 多头注意力机制

### 为什么 emb_dim 必须能被 n_heads 整除？

```
┌─────────────────────────────────────────────────────────────────────┐
│              多头注意力的维度分割                                   │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  emb_dim = 768, n_heads = 12                                       │
│  head_dim = 768 / 12 = 64 ✅                                       │
│                                                                     │
│  原始 Q 向量 (768 维):                                              │
│  ┌───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┬───┐                │
│  │   │   │   │   │   │   │   │   │   │   │   │   │                │
│  │64 │64 │64 │64 │64 │64 │64 │64 │64 │64 │64 │64 │                │
│  │   │   │   │   │   │   │   │   │   │   │   │   │                │
│  └───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┴───┘                │
│    ↓   ↓   ↓   ↓   ↓   ↓   ↓   ↓   ↓   ↓   ↓   ↓                  │
│   h1  h2  h3  h4  h5  h6  h7  h8  h9 h10 h11 h12                   │
│                                                                     │
│  每个头独立计算注意力，最后拼接回 768 维                            │
│                                                                     │
│  ════════════════════════════════════════════════════════════════  │
│                                                                     │
│  ❌ 错误配置: emb_dim=1280, n_heads=25                              │
│     head_dim = 1280 / 25 = 51.2 ← 不是整数！                       │
│     → 无法 reshape，AssertionError                                  │
│                                                                     │
│  ✅ 正确配置:                                                       │
│     emb_dim=1280, n_heads=20 → head_dim=64 ✅                      │
│     emb_dim=1600, n_heads=25 → head_dim=64 ✅                      │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

## 7. Scaling Law 与训练策略

### Scaling Law 核心公式

```
┌─────────────────────────────────────────────────────────────────────┐
│              Scaling Law: 不是无脑堆卡，是科学配方                  │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  核心公式 (幂律关系):                                               │
│                                                                     │
│  L(N) ∝ N^(-0.076)    参数量翻倍 → 损失下降 ~5%                    │
│  L(D) ∝ D^(-0.095)    数据量翻倍 → 损失下降 ~6.5%                  │
```

### The Bitter Lesson 核心观点

```
┌─────────────────────────────────────────────────────────────────────┐
│              苦涩的教训 (Rich Sutton, 2019)                         │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  核心论点:                                                          │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │  "从 70 年 AI 研究中学到的最大教训:                         │   │
│  │   利用计算能力的通用方法（搜索 + 学习）最终总是赢家"        │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                                                                     │
│  两种方法论的对立:                                                  │
│                                                                     │
│  ┌─────────────────────┐      ┌─────────────────────┐              │
│  │   人类知识方法      │  vs  │   通用计算方法      │              │
│  ├─────────────────────┤      ├─────────────────────┤              │
│  │ • 专家系统          │      │ • 搜索              │              │
│  │ • 特征工程          │      │ • 学习              │              │
│  │ • 手工规则          │      │ • 神经网络          │              │
│  ├─────────────────────┤      ├─────────────────────┤              │
│  │ 短期: ✅ 快速见效   │      │ 短期: ❌ 需要算力   │              │
│  │ 长期: ❌ 遇到瓶颈   │      │ 长期: ✅ 持续提升   │              │
│  └─────────────────────┘      └─────────────────────┘              │
│                                                                     │
│  历史案例:                                                          │
│  • 国际象棋: 40年知识编码 → 被"暴力搜索"打败                       │
│  • 围棋: "需要直觉" → 被 AlphaGo 打败                              │
│  • 语音: 语言学特征 → 被统计模型 → 被深度学习打败                  │
│  • 视觉: 手工特征(SIFT) → 被 CNN 打败                              │
│                                                                     │
│  "苦涩"在哪里:                                                      │
│  研究者多年积累的领域知识被证明不如"堆算力"有效                    │
│  → 心理上难以接受                                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```
