## 一、全文翻译

### 苦涩的教训

**Rich Sutton**
2019年3月13日

从 70 年的 AI 研究中可以读出的最大教训是：**利用计算能力的通用方法最终是最有效的，而且优势巨大**。其根本原因是摩尔定律，或者更准确地说，是单位计算成本持续指数级下降的普遍规律。

大多数 AI 研究是在假设智能体可用的计算能力恒定的前提下进行的（在这种情况下，利用人类知识将是提高性能的少数途径之一）。但是，在比典型研究项目稍长的时间尺度上，大规模的计算能力必然会变得可用。研究人员为了寻求短期内能产生差异的改进，试图利用他们对领域的人类知识。但**从长远来看，唯一重要的是对计算能力的利用**。

这两种方法不必相互对立，但在实践中它们往往是对立的。花在一种方法上的时间就是没花在另一种方法上的时间。人们对一种或另一种方法的投入存在心理承诺。而且，基于人类知识的方法往往会使方法复杂化，使其不太适合利用通用的计算方法。

AI 研究人员迟迟才学到这个苦涩教训的例子有很多，回顾一些最突出的例子是很有启发性的。

---

**在计算机国际象棋中**，1997 年击败世界冠军卡斯帕罗夫的方法是基于大规模深度搜索的。当时，大多数计算机国际象棋研究人员对此感到沮丧，因为他们一直追求的是利用人类对国际象棋特殊结构理解的方法。当一种更简单的、基于搜索的方法（配合专用硬件和软件）被证明更加有效时，这些基于人类知识的国际象棋研究人员并不是好的输家。他们说"暴力"搜索这次可能赢了，但这不是一种通用策略，而且这也不是人类下棋的方式。这些研究人员希望基于人类输入的方法获胜，当事与愿违时他们很失望。

---

**在计算机围棋中**，类似的研究进展模式出现了，只是推迟了 20 年。最初投入了大量精力来避免搜索，试图利用人类知识或游戏的特殊特征。但所有这些努力都被证明是无关紧要的，甚至更糟——一旦搜索被有效地大规模应用。同样重要的是利用自我对弈来学习价值函数（这在许多其他游戏甚至国际象棋中也是如此，尽管学习在 1997 年首次击败世界冠军的程序中没有发挥重要作用）。

自我对弈学习，以及一般的学习，就像搜索一样，它使大规模计算能够发挥作用。**搜索和学习是 AI 研究中利用大规模计算的两类最重要的技术**。在计算机围棋中，就像在计算机国际象棋中一样，研究人员最初的努力是利用人类理解（以减少搜索需求），只有在很久以后，通过拥抱搜索和学习才取得了更大的成功。

---

**在语音识别中**，1970 年代有一场由 DARPA 赞助的早期竞赛。参赛者包括许多利用人类知识的特殊方法——关于单词、音素、人类声道等的知识。另一边是更具统计性质的新方法，基于隐马尔可夫模型（HMMs）进行更多计算。同样，统计方法战胜了基于人类知识的方法。这导致了整个自然语言处理领域的重大变化，在几十年间逐渐发生，统计和计算开始主导该领域。深度学习在语音识别中的最新崛起是这一一致方向的最新一步。深度学习方法更少依赖人类知识，使用更多计算，结合在大型训练集上的学习，产生了显著更好的语音识别系统。

与游戏一样，研究人员总是试图制造按照他们认为自己的思维方式工作的系统——他们试图将这些知识放入系统中——但这最终被证明是适得其反的，是研究人员时间的巨大浪费。因为通过摩尔定律，大规模计算变得可用，并找到了利用它的方法。

---

**在计算机视觉中**，有类似的模式。早期方法将视觉概念化为搜索边缘、广义圆柱体或 SIFT 特征。但今天所有这些都被抛弃了。现代深度学习神经网络只使用卷积和某些类型的不变性概念，表现得更好。

---

这是一个重大教训。作为一个领域，我们仍然没有彻底学会它，因为我们还在继续犯同样的错误。要看清这一点并有效抵制它，我们必须理解这些错误的吸引力。**我们必须学会这个苦涩的教训：将我们认为自己思考方式内建到系统中，从长远来看是行不通的。**

苦涩的教训基于以下历史观察：
1. AI 研究人员经常试图将知识内建到他们的智能体中
2. 这在短期内总是有帮助的，而且让研究人员个人感到满意
3. 但从长远来看，它会停滞不前，甚至抑制进一步的进展
4. 突破性进展最终来自于相反的方法——基于通过搜索和学习来扩展计算

最终的成功带有苦涩的味道，而且往往没有被完全消化，因为这是对一种受青睐的、以人类为中心的方法的胜利。

---

从苦涩教训中应该学到的一件事是**通用方法的强大力量**——即使在可用计算变得非常大时，也能随着增加的计算而持续扩展的方法。以这种方式可以任意扩展的两种方法是 **搜索（Search）** 和 **学习（Learning）**。

从苦涩教训中应该学到的第二个一般性观点是：**心智的实际内容是极其复杂的，无法简化的**。我们应该停止试图找到关于心智内容的简单思考方式，比如关于空间、物体、多智能体或对称性的简单思考方式。所有这些都是任意的、内在复杂的外部世界的一部分。它们不应该被内建进去，因为它们的复杂性是无穷的。相反，**我们应该只内建能够发现和捕获这种任意复杂性的元方法（meta-methods）**。

这些方法的关键在于它们能够找到好的近似，但搜索它们应该由我们的方法完成，而不是由我们自己完成。**我们想要的是能够像我们一样发现的 AI 智能体，而不是包含我们已经发现的东西的 AI 智能体**。将我们的发现内建进去只会使我们更难看清发现过程是如何完成的。

## 二、核心概念解析

### 2.1 核心论点（The Central Thesis）

```
┌─────────────────────────────────────────────────────────────────┐
│                      苦涩教训的核心                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   短期有效 ≠ 长期有效                                           │
│                                                                 │
│   人类知识内建 → 短期见效快 → 长期遇到天花板                     │
│   通用计算方法 → 短期见效慢 → 长期持续提升                       │
│                                                                 │
│   最终赢家：Search（搜索）+ Learning（学习）                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 两种方法论的对立

| 维度 | 人类知识方法 | 通用计算方法 |
|------|-------------|-------------|
| **核心思想** | 将人类专家知识编码进系统 | 让系统通过计算自己发现 |
| **短期效果** | ✅ 快速见效 | ❌ 需要大量计算才有效 |
| **长期效果** | ❌ 遇到瓶颈 | ✅ 持续提升 |
| **扩展性** | 受限于人类理解 | 随计算量增加而提升 |
| **研究者感受** | 满足感强（"我的知识有用"） | 满足感弱（"只是堆算力"） |
| **代表** | 专家系统、特征工程、规则 | 搜索、学习、神经网络 |

### 2.3 关键术语解释

#### **搜索（Search）**

在可能的解空间中系统地探索，寻找最优或足够好的解。

**类比**：就像在迷宫中尝试所有路径，而不是依赖"左转右转"的规则。

**例子**：
- 国际象棋的 minimax 搜索
- AlphaGo 的蒙特卡洛树搜索（MCTS）
- 大语言模型的 beam search

$$
\text{Search}: \quad \arg\max_{a \in \mathcal{A}} Q(s, a) \quad \text{（在动作空间中搜索最优动作）}
$$

#### **学习（Learning）**

从数据中自动发现模式和规律，无需显式编程。

**类比**：就像孩子通过大量接触语言自然学会说话，而不是背语法规则。

**例子**：
- 监督学习（从标注数据学习）
- 强化学习（从奖励信号学习）
- 自监督学习（从数据本身学习）

$$
\text{Learning}: \quad \theta^* = \arg\min_\theta \mathcal{L}(\theta; \mathcal{D}) \quad \text{（从数据中学习参数）}
$$

#### **元方法（Meta-methods）**

不是解决具体问题的方法，而是**发现解决方法的方法**。

**类比**：
- 具体方法 = 一道菜的食谱
- 元方法 = 如何创造新食谱的方法论

**例子**：
- 梯度下降（学习如何调整参数）
- 神经架构搜索（学习如何设计网络）
- 元学习（学习如何学习）

## 三、历史案例分析

### 3.1 计算机国际象棋

```
时间线：
1950s-1990s: 研究人员努力编码国际象棋知识（开局库、残局规则、位置评估...）
1997: Deep Blue 击败卡斯帕罗夫 → 主要靠暴力搜索 + 专用硬件
2017: AlphaZero 击败 Stockfish → 纯自我对弈学习，零人类知识

教训：
  人类专家花了 40 年编码国际象棋知识
  → 被"暴力"搜索打败
  → 又被纯学习方法打败
```

### 3.2 计算机围棋

```
时间线：
1990s-2015: 研究人员认为围棋太复杂，搜索空间太大，必须用人类知识
           → 开发了大量特征：眼位、气、势力范围...
2016: AlphaGo 击败李世石 → 深度学习 + MCTS
2017: AlphaGo Zero → 零人类知识，纯自我对弈

教训：
  人类认为"围棋需要直觉，不能暴力搜索"
  → 被 搜索 + 学习 的组合打败
```

### 3.3 语音识别

```
时间线：
1970s: 基于语言学知识（音素、发音规则、声道模型...）
1980s-2000s: 统计方法（HMM）胜出
2010s: 深度学习胜出（端到端，更少人工特征）

教训：
  语言学家花了几十年研究语音的"科学规律"
  → 被统计模型打败
  → 又被端到端神经网络打败
```

### 3.4 计算机视觉

```
时间线：
1970s-2000s: 手工特征（边缘检测、SIFT、HOG...）
2012: AlexNet 在 ImageNet 上大获全胜
2020s: Vision Transformer、大规模预训练

教训：
  视觉研究者花了 40 年设计"好的特征"
  → 被端到端学习的卷积网络打败
  → 特征工程几乎消失
```

## 四、对偶概念与扩展

### 4.1 苦涩教训的对立面：甜蜜的捷径？

有人提出了与苦涩教训相对的观点：

| 苦涩教训 | "甜蜜的捷径"（反对观点） |
|---------|------------------------|
| 通用方法 + 大计算最终胜出 | 归纳偏置（Inductive Bias）加速学习 |
| 不要内建人类知识 | 适当的先验知识帮助泛化 |
| 让系统自己发现一切 | 人类知识可以作为"warm start" |

**例子**：卷积神经网络（CNN）内建了"平移不变性"假设，这是人类知识，但非常有效。

**Sutton 的回应**：CNN 的成功恰恰是因为它比手工特征更"通用"，而且它的优势随计算量增加而增加。

### 4.3 Scaling Laws（扩展定律）

苦涩教训在 2020 年代得到了更精确的数学表达：

$$
L(C) = \left(\frac{C_0}{C}\right)^\alpha
$$

其中：
- $L$ = 损失（越低越好）
- $C$ = 计算量
- $\alpha \approx 0.05$ (对于语言模型)

**含义**：损失与计算量呈幂律关系，更多计算 = 更好性能，没有明显的天花板。

**这正是苦涩教训的量化验证**。

### 4.4 你可能"不知道自己不知道"的

#### **No Free Lunch 定理**

数学上证明：没有一种学习算法在所有问题上都是最优的。

**但是**：现实世界的问题有共同的结构（物理规律、因果关系...），所以通用方法可以有效。

#### **Occam's Razor vs Bitter Lesson**

| Occam's Razor | Bitter Lesson |
|---------------|---------------|
| 简单的模型更好 | 更大的模型更好 |
| 偏好简洁 | 偏好计算 |

**调和**：大模型可以学到"有效的简洁性"——它们可能在内部发现了简洁的规律。

#### **Chinchilla vs Bitter Lesson**

Chinchilla 论文（2022）发现：在固定计算预算下，应该平衡模型大小和数据量。

这**不否定**苦涩教训，而是**精化**了它：
- 苦涩教训：更多计算 → 更好
- Chinchilla：计算应该**如何分配**到模型和数据

## 五、对 LLM 时代的启示

### 5.1 GPT 系列是苦涩教训的最佳证明

```
GPT-1 (2018):  117M 参数
GPT-2 (2019):  1.5B 参数
GPT-3 (2020):  175B 参数
GPT-4 (2023):  ~1.8T 参数（传闻）

方法几乎没变（Transformer + 自回归）
主要变化：更多计算、更多数据、更大模型
结果：能力的涌现
```

### 5.2 对你学习 LLM 的启示

| 苦涩教训的启示 | 对学习者的意义 |
|---------------|---------------|
| 通用方法最终胜出 | 学好 Transformer、反向传播等通用概念 |
| 搜索和学习是关键 | 理解 SGD、采样、搜索算法 |
| 不要过度依赖直觉 | 相信实验结果，不要先入为主 |
| 扩展性很重要 | 关注方法能否随计算量提升 |

### 5.3 苦涩教训的"苦涩"之处

```
对研究者：
  - 多年积累的领域知识可能变得无用
  - "我懂 X 领域" 不如 "我会训练大模型"
  - 人类直觉被机器超越

对人类：
  - AI 不是"像人一样思考"
  - AI 用完全不同的方式达到甚至超越人类能力
  - 这让人感到不安和失落

这就是"苦涩"的来源。
```

## 六、总结

### 核心观点

> **从 70 年 AI 研究中学到的最大教训：利用计算能力的通用方法（搜索 + 学习）最终总是赢家。**

### 三句话总结

1. **短期**：内建人类知识见效快，让研究者有成就感
2. **长期**：通用的搜索和学习方法会持续改进，最终超越
3. **启示**：构建能够**发现**的系统，而不是构建包含**发现成果**的系统

### 一个类比

```
人类知识方法 = 给孩子一本百科全书，让他背下来
通用计算方法 = 给孩子好奇心和学习能力，让他自己探索世界

前者很快能答题，但受限于百科全书的内容
后者一开始很慢，但最终能超越百科全书
```

### Rich Sutton 的终极洞见

> **"我们想要的是能够像我们一样发现的 AI 智能体，而不是包含我们已经发现的东西的 AI 智能体。"**

这句话预言了 ChatGPT 时代的到来——一个通过大规模学习而非人工规则，展现出惊人能力的系统。
