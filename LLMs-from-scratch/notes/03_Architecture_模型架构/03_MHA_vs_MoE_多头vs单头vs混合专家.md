# 多头注意力 vs 单头注意力 vs MoE：深度对比

> 本文从设计哲学、计算原理、类比联想三个维度，深入剖析这三种架构的异同。

---

## 1. 多头注意力（Multi-Head Attention）的好处

### 1.1 核心思想

多头注意力的核心洞见是：**不同的"头"可以关注不同的语义子空间**。

原论文 *"Attention is All You Need"* (Vaswani et al., 2017) 的原话：

> "Multi-head attention allows the model to **jointly attend to information from different representation subspaces** at different positions."

### 1.2 数学表达

**单头注意力**：

$$
\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V
$$

其中 $d_k$ 是 key 的维度。

**多头注意力**：

$$
\text{MultiHead}(Q, K, V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)W^O
$$

$$
\text{head}_i = \text{Attention}(QW_i^Q, KW_i^K, VW_i^V)
$$

其中：
- $W_i^Q \in \mathbb{R}^{d_{model} \times d_k}$
- $W_i^K \in \mathbb{R}^{d_{model} \times d_k}$
- $W_i^V \in \mathbb{R}^{d_{model} \times d_v}$
- $W^O \in \mathbb{R}^{hd_v \times d_{model}}$
- $h$ 是头的数量
- 通常 $d_k = d_v = d_{model} / h$

### 1.3 多头注意力的具体好处

#### 好处 1：捕获多种语义关系

```
句子: "The cat sat on the mat because it was tired"

头 1 (语法头): "it" → "cat" (指代关系)
头 2 (位置头): "it" → "mat" (近距离名词)
头 3 (语义头): "tired" → "sat" (动作-状态关系)
```

**类比**：就像用多个相机从不同角度拍摄同一场景，每个相机捕捉不同的信息。

#### 好处 2：降低注意力"过度集中"的风险

单头注意力可能会过度关注某个位置，导致信息瓶颈。多头相当于"分散投资"，降低单点失败的风险。

$$
\text{单头}: \quad p = \text{softmax}(q \cdot K^T) \quad \text{(一个概率分布)}
$$

$$
\text{多头}: \quad p_1, p_2, \ldots, p_h \quad \text{(多个独立的概率分布)}
$$

#### 好处 3：计算效率

令人惊讶的是，**多头注意力和单头注意力的计算量几乎相同**！

| 架构 | 参数量 | FLOPs |
|------|--------|-------|
| 单头 ($d_{model}=512$) | $3 \times 512^2 = 786K$ | $O(n^2 \cdot d)$ |
| 多头 (8头, $d_k=64$) | $3 \times 8 \times 512 \times 64 = 786K$ | $O(n^2 \cdot d)$ |

**关键洞见**：多头不是"更贵"的设计，而是**用相同计算量换取更丰富的表达能力**。

#### 好处 4：训练稳定性

多头提供了某种程度的"集成学习"效果，使梯度更加稳定。

## 2. 单头注意力的特点与场景

### 2.1 单头的潜在好处

| 特点 | 说明 |
|------|------|
| **更简单** | 实现简单，调试容易 |
| **更统一的表示** | 所有信息在同一个语义空间中交互 |
| **特定任务可能更好** | 当任务只需要一种注意力模式时 |

### 2.2 单头的研究发现

Michel et al. (2019) 的论文 *"Are Sixteen Heads Really Better than One?"* 发现：

> 在某些层中，移除大部分头对性能影响很小，有时甚至**只保留一个头**也能保持 95% 以上的性能。

这说明：
1. 不是所有层都需要多头
2. 多头的"冗余"可能是训练的副产品
3. 不同层的头有不同的重要性

### 2.3 单头的适用场景

```python
# 适合单头的场景
scenarios_for_single_head = [
    "资源极度受限的边缘设备",
    "任务本身只需要一种注意力模式",
    "作为研究 baseline",
    "知识蒸馏后的轻量模型"
]
```

## 3. 多头注意力 vs MoE：深度对比

### 3.1 核心设计哲学

| 维度 | 多头注意力 (MHA) | 混合专家 (MoE) |
|------|------------------|----------------|
| **核心思想** | 分割语义空间 | 分割参数空间 |
| **激活方式** | **全部激活** | **稀疏激活** |
| **分工依据** | 固定的空间划分 | 动态的路由决策 |
| **应用位置** | 注意力层 | 通常在 FFN 层 |

### 3.2 相似之处（关键洞见）

#### 相似点 1：分而治之（Divide and Conquer）

两者都体现了**专业化分工**的思想：

```
多头注意力:
┌─────────────────────────────────────────────┐
│  输入 embedding (768维)                      │
└─────────────────────────────────────────────┘
           │ 均匀分割
           ▼
┌─────┐ ┌─────┐ ┌─────┐    ┌─────┐
│头 1 │ │头 2 │ │头 3 │... │头 12│  (每头 64 维)
│语法 │ │指代 │ │语义 │    │位置 │
└─────┘ └─────┘ └─────┘    └─────┘
           │ 拼接
           ▼
┌─────────────────────────────────────────────┐
│  输出 (768维)                                │
└─────────────────────────────────────────────┘


MoE:
┌─────────────────────────────────────────────┐
│  输入 token                                  │
└─────────────────────────────────────────────┘
           │ 路由器决策
           ▼
┌─────────┐ ┌─────────┐    ┌─────────┐
│专家 1   │ │专家 2   │... │专家 N   │
│数学知识 │ │语言知识 │    │代码知识 │
└─────────┘ └─────────┘    └─────────┘
     ✗          ✓               ✗      (稀疏激活)
           │ 加权组合
           ▼
┌─────────────────────────────────────────────┐
│  输出                                        │
└─────────────────────────────────────────────┘
```

#### 相似点 2：并行处理

两者都支持高效的并行计算：
- MHA：每个头可以并行计算
- MoE：每个专家可以并行计算

#### 相似点 3：多样性学习

两者都期望不同组件学到不同的"知识"：
- MHA：不同头学习不同的注意力模式
- MoE：不同专家学习不同的领域知识

### 3.3 关键区别

#### 区别 1：激活模式（最核心的区别）

```python
# 多头注意力：全部激活
def multi_head_attention(x, heads):
    outputs = []
    for head in heads:
        outputs.append(head(x))  # 每个头都计算
    return concat(outputs)  # 全部拼接

# MoE：稀疏激活
def moe_forward(x, experts, router, top_k=2):
    weights = router(x)  # 计算路由权重
    top_k_experts = select_top_k(weights, k=top_k)  # 只选 top-k 个
    output = 0
    for expert, weight in top_k_experts:
        output += weight * expert(x)  # 只有被选中的专家计算
    return output
```

**数学表达**：

MHA:
$$
\text{Output} = \text{Concat}(h_1, h_2, \ldots, h_H) \cdot W^O \quad \text{(所有 } h_i \text{ 都计算)}
$$

MoE:
$$
\text{Output} = \sum_{i \in \text{TopK}} g_i \cdot E_i(x) \quad \text{(只有 TopK 个专家计算)}
$$

其中 $g_i = \text{Softmax}(\text{Router}(x))_i$

#### 区别 2：计算成本 vs 参数量

| 指标 | 多头注意力 | MoE |
|------|------------|-----|
| 参数量 | 中等 | **极大** (可达万亿) |
| 每次前向计算量 | 使用全部参数 | 只使用部分参数 |
| 扩展方式 | 加深/加宽 | 增加专家数量 |

**关键洞见**：MoE 的设计目标是 **"用参数换效率"**——拥有巨大的参数量，但每次推理只激活一小部分。

Switch Transformer (2021) 的数据：
- **1.6 万亿参数**
- 每次前向只激活 **约 1%** 的参数
- 相同计算量下，性能比 T5-XXL **提升 4 倍**

#### 区别 3：分工机制

```
多头注意力的分工：
  - 静态的、预定义的空间划分
  - head_dim = emb_dim / n_heads (固定)
  - 每个头总是处理相同维度的数据
  - 类比：流水线上的固定工位

MoE 的分工：
  - 动态的、输入决定的路由
  - 路由器根据输入决定派给哪个专家
  - 不同输入可能激活不同专家
  - 类比：医院的分诊台，根据症状派给不同科室
```

## 4. 扩展概念：你可能"不知道自己不知道"的

### 4.1 Multi-Query Attention (MQA) 和 Grouped-Query Attention (GQA)

这是多头注意力的变体，用于优化推理效率：

```
标准 MHA:    每个头有独立的 Q, K, V
Multi-Query: 所有头共享 K, V，只有 Q 独立
Grouped-Query: 将头分组，组内共享 K, V

参数量: MHA > GQA > MQA
推理速度: MQA > GQA > MHA
性能: MHA ≈ GQA > MQA
```

LLaMA 2 使用 GQA，Falcon 使用 MQA。

### 4.2 为什么 MoE 通常用在 FFN 而不是 Attention？

```python
# FFN 的参数量远大于 Attention
# GPT-2 Small 的一个 Transformer 块:
attention_params = 4 * emb_dim^2  # Q, K, V, O 投影
ffn_params = 8 * emb_dim^2        # 两个线性层 (4倍扩展)

# FFN 参数是 Attention 的 2 倍！
# 所以 MoE 用在 FFN 上效益更大
```

### 4.3 Head 的"专业化"是自动学习的

研究发现，多头注意力的不同头会**自动**学习到不同的模式：

| 头类型 | 学到的模式 |
|--------|-----------|
| 位置头 | 关注相邻位置 |
| 语法头 | 关注句法依赖 |
| 罕见词头 | 特别关注低频词 |
| 分隔符头 | 关注句子边界（如句号）|

这种专业化是**涌现**的，不是人为设计的。

### 4.4 稀疏注意力：MHA 和 MoE 思想的融合

最新的研究尝试将 MoE 的稀疏思想应用到注意力：

- **Sparse Transformer**: 不计算所有位置对的注意力
- **Longformer**: 局部注意力 + 全局注意力
- **BigBird**: 随机注意力 + 局部注意力 + 全局注意力

## 6. 总结

| 维度 | 多头注意力 | 单头注意力 | MoE |
|------|-----------|-----------|-----|
| **核心思想** | 分割语义子空间 | 统一的注意力 | 分割参数空间 |
| **激活模式** | Dense (全部) | Dense | Sparse (部分) |
| **计算成本** | 中等 | 较低 | 可控 (与稀疏度相关) |
| **参数量** | 中等 | 较低 | 极大 |
| **适用场景** | 通用 | 简单任务/资源受限 | 超大规模模型 |
| **分工方式** | 静态空间划分 | 无分工 | 动态路由 |

### 核心洞见

1. **多头注意力**的本质是**用相同计算量换取更丰富的表达**
2. **MoE**的本质是**用更多参数换取计算效率**（每次只用部分参数）
3. 两者都体现了"**专业化分工**"的设计哲学
4. 最大的区别：多头是**Dense**（全部激活），MoE是**Sparse**（部分激活）

### 类比总结

```
多头注意力 ≈ 多个摄影师同时从不同角度拍摄（全员出动）
单头注意力 ≈ 一个摄影师用广角镜头拍摄（一人包揽）
MoE ≈ 大型医院的分诊系统（根据症状派专家，不是所有专家都出诊）
```
