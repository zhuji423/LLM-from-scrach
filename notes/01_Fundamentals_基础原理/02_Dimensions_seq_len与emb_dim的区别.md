# seq_len vs emb_dim：两个完全不同的概念

## 澄清

```python
batch_size, seq_len = in_idx.shape
#                     [2, 1024]
#                      ↑   ↑
#                  batch  序列长度（有多少个 token）
```

| 变量 | 含义 | 典型值 | 例子 |
|------|------|--------|------|
| `seq_len` | 序列长度（输入了多少个 token） | ≤ 1024 | "Hello world" → 2 个 token |
| `emb_dim` | 嵌入维度（每个 token 的向量维度） | 768 | 每个 token 是 768 维向量 |

**这是两个完全不同的概念！**

---

## 维度流转

```
输入 in_idx:           [batch_size, seq_len]
                       [2, 1024]           ← seq_len 在这里

经过 tok_emb:          [batch_size, seq_len, emb_dim]
                       [2, 1024, 768]       ← emb_dim 在这里

经过 Transformer:      [batch_size, seq_len, emb_dim]
                       [2, 1024, 768]

经过 out_head:         [batch_size, seq_len, vocab_size]
                       [2, 1024, 50257]
```

---

## 类比

```
seq_len = 一句话有多少个词（横向，时间维度）
emb_dim = 每个词的描述有多详细（纵向，特征维度）
```

**例子**：

```
"我 爱 北京 天安门"

seq_len = 4（4 个词）
emb_dim = 768（每个词用 768 个数字描述）

┌─────┬─────┬─────┬─────┐
│  我 │  爱 │北京 │天安门│  ← seq_len = 4
├─────┼─────┼─────┼─────┤
│ 768 │ 768 │ 768 │ 768 │  ← 每个词 emb_dim = 768
│  维 │  维 │  维 │  维 │
└─────┴─────┴─────┴─────┘
```

---

## 关键区分

| | seq_len | emb_dim |
|--|---------|---------|
| **是什么** | 序列有多长 | 每个位置的向量有多大 |
| **维度** | 第 2 维 | 第 3 维 |
| **来源** | 输入决定（≤ context_length） | 模型架构决定（固定） |
| **类比** | 一句话有几个词 | 每个词的"内涵"有多丰富 |
| **典型值** | 1~1024（可变） | 768（固定） |

---

## 一句话总结

> **seq_len 是"有多少个 token"（≤ context_window），768 是"每个 token 的向量维度"（emb_dim），两者不是一回事。**
